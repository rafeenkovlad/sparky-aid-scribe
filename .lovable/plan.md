## Цель
В карточке элемента в шаге «Осмотр» сделать переключатель «Мелкие» / «Серьёзные» обычными вкладками: клик по вкладке мгновенно показывает теги соответствующего раздела, не дожидаясь, пока вердикт «успеет» переключиться. «Без замечаний» остаётся отдельной кнопкой вердикта (скрывает теги).

## Что меняется
Файл: `src/components/carreports/InspectionChipsCard.tsx`

1. В `ElementBlock` ввести локальное состояние `activeTab: "minor" | "serious"`.
   - Инициализация: из `verdict` (если `serious` → `serious`, иначе `minor`).
   - При смене `section.snake` / элемента — сбрасывать на значение, выведенное из текущего вердикта.

2. Сегмент вердикта переписать как комбинированный контрол:
   - Кнопка «Без замечаний» — по‑прежнему вызывает `onSetVerdict("ok")`.
   - Кнопки «Мелкие» / «Серьёзные» работают как вкладки: сразу меняют `activeTab` и одновременно вызывают `onSetVerdict(...)` (поведение сохранения вердикта в данных не ломаем).
   - Подсветка активной вкладки — по `activeTab` (а не по `verdict`), чтобы клик давал моментальный визуальный отклик даже если `derivedVerdict` ещё не обновился.

3. В `TagsArea` заменить расчёт `primaryIsSerious` с `verdict === "serious"` на проп `activeTab === "serious"`.
   - Скрывать теги, только если `verdict === "ok"` (как сейчас).
   - Передавать `activeTab` из `ElementBlock`.

4. `activeBucket` для нового тега и плейсхолдер инпута — тоже по `activeTab`.

5. Существующий `verdictOverride` оставляем — он по‑прежнему нужен, чтобы клик сразу подсвечивал вердикт. Просто `activeTab` теперь дополнительно гарантирует, что список тегов перерисовывается мгновенно.

## Чего НЕ трогаем
- Поведение `onSetVerdict`, `onToggleTag`, `onAddPendingTag` и оркестратор.
- `SectionPickerCard`, селектор разделов, селектор элементов.
- Логику «Без замечаний» (она по‑прежнему очищает теги и скрывает блок).
