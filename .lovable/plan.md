## Что наследуем

Из шага car/characteristics уже работает:
1. Уточняющие вопросы — отдельным сообщением (`tailLine` пушится после карточки).
2. Чипы вариантов — кликабельные плашки, значения «прокидываются» в поле ввода (нередактируемые).
3. Крупные изображения скрыты в уже сформированной карточке (показываются только миниатюры).
4. Дата осмотра — только в активной карточке.

Сейчас на шагах **Осмотр / Тест-драйв / Итог** этого нет: оркестратор склеивает «карточку» + «продолжайте…» в один `reply`, чипов не отдаёт, всё уточнение — внутри текста.

## Изменения

### 1. `src/lib/carreports/orchestrator.ts`

**inspection (66–214):**
- Убрать из `reply` строку «Продолжайте по этой зоне…» — она поедет универсальным `tailLine` из `ChatApp`.
- Возвращать `chips` со списком элементов текущей зоны (`section.elements`) с группой `inspection-element` + чипы серьёзных/несерьёзных тегов (группа `inspection-tag`). `single: true`, чтобы клик подставлял значение в инпут как готовую фразу («Заднее левое крыло — скол лака»).
- Кнопка зоны остаётся отдельным UI (как сейчас).

**testDrive (217–268):**
- Убрать хвост «Дополните, либо «Всё верно, далее»» из `reply`.
- Чипы: системы (двигатель/КПП/руль/подвеска/тормоза) × состояние («работает корректно», «есть замечания»), плюс «Тест-драйв не проводился».

**result (271–307):**
- Убрать хвост «Дополните или нажмите …» из `reply`.
- Чипы-шаблоны: «✅ Рекомендую к покупке», «⚠️ Покупать с торгом», «❌ Не рекомендую», «📝 Резюме осмотра», «🔧 Список работ перед покупкой».

### 2. `src/lib/carreports/progress.ts` (`nextMissingPrompt`)

Добавить осмысленные follow-up подсказки для шагов inspection / testDrive / result, чтобы универсальный `tailLine` показывал что-то полезное вместо «✅ Шаг заполнен» сразу.
- inspection: «Опишите ещё элементы зоны или выберите следующую зону / нажмите чип элемента».
- testDrive: «Опишите поведение машины в движении или выберите системы ниже».
- result: «Сформулируйте резюме и вердикт — или выберите шаблон ниже».

### 3. `src/components/carreports/ChatApp.tsx`

Универсальная логика рендера уже подходит, нужны точечные правки:
- В обработчике клика по чипу для групп `inspection-element` / `inspection-tag` / `testDrive-*` / `result-template` подставлять `chip.value` в инпут без редактирования (как сейчас для generation-чипов).
- В фильтре `small`-вложений (1010-1020) расширить исключение крупных картинок на новые `kind`-ы, если появятся (для inspection/result картинок нет — изменений не нужно).
- Убедиться, что `optionsStep`/`lastOptionsMsgId` корректно отмечает интерактивной только последнюю карточку каждого шага (уже работает универсально).

### 4. `src/lib/carreports/types.ts`

Расширить тип `ChatChip['group']` новыми значениями: `"inspection-element" | "inspection-tag" | "testDrive-system" | "result-template"`.

## Технические детали

- Все правки — на клиенте, без миграций и без новых серверных вызовов.
- Каталог тегов уже подгружается в осмотре (`loadSectionTags`), используем тот же список для чипов.
- Чипы делаем `single: true` — клик добавляет одну фразу в инпут, остальные чипы остаются доступны до отправки.

## Открытые вопросы

1. На осмотре чипы тегов — показывать **все** теги секции или только «частые» (топ-10)? Полный список может быть длинным.
2. На тест-драйве чипы — это пары «система + статус» (двойной клик) или сразу готовые фразы «Двигатель работает корректно» / «У КПП есть замечания»? Предлагаю готовые фразы — проще кликать.
